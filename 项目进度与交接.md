## 一、项目基本信息

Lan Code 是一个本地优先、客户端无关的 AI 编程 Agent。核心使用 Rust，
桌面端使用 Tauri + React。未来计划支持 CLI、桌面端、VS Code、JetBrains 和 Web。

核心原则：

1. Agent 真相属于 core，不属于桌面 UI。
2. 模型只能提出工具调用，权限与执行由 core 控制。
3. 会话、工具、审批和事件必须可持久化、可恢复。
4. 默认保护用户文件；高风险命令必须明确授权。
5. 所有面向用户和开发交接的文档优先使用中文。

主要目录：

| 路径 | 作用 |
| --- | --- |
| `crates/lan-protocol` | 跨客户端协议类型 |
| `crates/lan-core` | Agent loop、模型适配、工具、权限、存储 |
| `crates/lan-daemon` | JSONL daemon，为未来多客户端提供边界 |
| `crates/lan-cli` | CLI 客户端 |
| `apps/desktop` | Tauri + React 桌面端 |
| `research/repos` | 18 个参考项目的浅克隆，不作为依赖 |
| `docs/research-analysis.zh-CN.md` | 参考项目研究结论 |

## 四、本轮用户反馈与决策

用户反馈：

1. Windows 安装器触发 SmartScreen。
2. 长任务生成了一部分文件，但最后因超过 12 轮被判失败。
3. 功能太少，需要多模型、模型可用性测试、项目管理、丰富设置、删除会话。
4. 默认数据目录应为用户主目录下的 `.lancode`，并允许用户自行选择。
5. 用户接受 API Key 明文保存在数据目录配置文件中。
6. 希望加入 Token 计费和更多 Kun 类能力。

当前产品决策：

- 默认数据目录改为 `~/.lancode`。
- 支持用户选择自定义数据目录。
- API Key 可明文写入用户选择的数据目录；UI 必须明确提示风险。
- Provider 先统一走 OpenAI-compatible 协议，并提供常用预设；之后再增加
  Anthropic/Gemini 原生能力适配。
- SmartScreen 的根本解决方案是购买并配置 Windows 代码签名证书。未签名版本只能
  提供 SHA-256、发布说明和风险提示，不能消除 SmartScreen。
